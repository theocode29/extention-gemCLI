### 10.2 Flux de Données et Séquencement

**Séquence Type d'Itération:**

1. Gemini lit `.gemini_tasks.json` via `read_task_state()`
2. Gemini appelle `scaffold_project()` (si nouveau projet)
3. Pour chaque tâche:
   - Gemini crée un **plan détaillé** puis le suit
   - Pour modifications: `read_file()` → analyse → `edit_file_replace()` ou `edit_json_value()`
   - Après chaque modif: Git commit auto + `run_smart_linter()`
   - Si librairie needed: `list_capabilities()` → `request_capability()` → `link_library()`
   - Si blocage: `request_human_review()`
   - Unease retourne `rollback_last_action()`

### PHASE 3 : LE ROUTEUR DE LIBRARIES (Semaine 4)

**Objectif** : Système modulaire de librairies avec scoring et injection auto.

**Sprint 3.1 : Schéma `.libmanifest.json` (Jours 1-2)**
- [ ] Définir schéma JSON (avec Zod ou validation manuelle):
  ```typescript
  interface LibManifest {
    name: string;                   // "Bookshelf"
    namespace: string;              // "bs"
    version: string;                // "1.2.0"
    description?: string;
    requires_load_tag?: string;     // "bs:load" (default: "{namespace}:load")
    requires_tick_tag?: string;     // "bs:tick" (default: "{namespace}:tick")
  ```

- [ ] Créer `src/subsystems/LibraryRegistry.ts`:

  - `loadManifest(path)` : parse + valide schéma

    ```typescript

    ```
  - Charge snippets `.md` en mémoire (ou chemins relatifs)
  - `getCapabilities()` : retourne liste dédoublonnée

  - Scan 2 manifestes (Bookshelf, GunCore)
  - Trouve capability "raycast" → retourne GunCore si tags["weapon"]
  - Gère libs sans capabilities (ignore)

**Sprint 3.2 : Algorithme de Scoring (Jours 3-4)**
- [ ] `CapabilityRouter.ts`:
  - `score(lib: LibManifest, capability: string, contextTags: string[]): number`
    * Score = `tag_matches_count * 2` (tags exacts)
    * Bonus: capability mentionné dans description +1

- **Économie tokens**: Snippets < 2KB, Linter retourne < 10 warnings typiques

### 15.4 Risques & Mitigations

| Risque | Impact | Probabilité | Mitigation |
|--------|--------|------------|------------|
| edit_file_replace trop fragile (indentation) | Bloquant | Haute | Validation pré-snip: comparer avec `read_file` output, suggestion précis |
| Spyglass CLI incomplet/bugs | Bloquant | Moyenne | Fallback linter simple (regex patterns) si Spyglass fail |
| Gemini génère code invalide malgré linter | Bloquant | Moyenne | HITL obligatoire avant première exécution Minecraft |
| Librairie introuvable | Majeur | Basse | `request_capability` retourne "aucun match" → HITL |
| Fuite mémoire snippets | Majeur | Moyenne | Cache avec TTL 1h, limite taille cache 50MB |
